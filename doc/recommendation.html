Topcoat - Design Recommendation

Introduction

When a developer or designer decides to use Topcoat, we can give them a hand with usability and style with patterns built on three elements:

  • Element style
  • Colour Palette
  • Typography

Directions

Direction One - recommended

After the initial survey we explored ways to adapt the desktop TopCoat patterns to a mobile environment. We modified the desktop version to meet some of these findings, this way the mobile version would differ less from it's desktop companion. The changes made to TopCoat averall were meant to go less skumorphic yet still invite the touch/click.

  • Default sizes of elements were increased by approximately a factor of 4.
  • All 1px line thicknesses were increased to 2px to ensure a crisp appearance on retina, and non-retina displays alike.

Direction Two

The second direction we explored goes further in some of the findings of the design survey. Button styles that are even less skeuomorphic than option 1, yet still hold affordance, reduce noise and give the style a cleaner look. More dynamic typography and increased white-space around elements increases hierarchy.

This direction relies on transparencies and shadows which take on the underlying background colour. Inspired by the ‘Quiet’ style from the TopCoat Desktop Kit, it affords the developer a language that is strong, yet unobtrusive.

Colour Palettes

Background and Proposal

Building our colour pallets from a single primary colour chosen by developers allows us to balance a personalized theme with beautiful usability.

Palette Composition

The palette would be made up of the following swatches:

  1. Primary
  2. Secondary
  3. Quiet (Semi-Transparent)
  4. Neutral #1 (light grey)
  5. Neutral #2 (dark grey)

Palette Creation

Adobe's Kuler and similar tools, allow us to use simple rules to reliably create beautiful and usable colour pallettes. We recommend using a monochromatic or triad palette.

Resulting palettes allow us to suggest specific uses for the colours derived from them. Since we know some of the qualities of the derivative colours, we can design for usability.

Type

Background and Proposal

Topcoat is currently using the open-source Adobe Source Sans typeface, designed for both short UI labels and longer passages of text on screen and in print [2]. A beautiful and effective type family, Source Sans offers a range of weights that aren't taken advantage of in the desktop TopCoat patterns. We suggest:

  • Employ a more dynamic range of type styles to create hierarchy and improve fit on platforms that are more type driven.
  • Provide some built-in scales to help users select an appropriate line-height as well as font size, weight, and style, giving a helping hand in terms of creating hierarchy [3][4].
  • Exploring how vertical rhythm can be established using baseline grids or padding around selected elements (for example, a CTA button) [5].

Type is often an overlooked factor in terms of usability, and building these tools into Topcoat will mean that each app will contain some basic typographic styles to enhance experience on mobile devices.
